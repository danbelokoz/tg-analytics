# TG Jobs — агрегатор вакансий из Telegram-каналов

> Эволюция проекта: из дашборда аналитики каналов (`index.html`) — в продукт для
> поиска работы. Аналитика остаётся «движком под капотом» (доверие к источникам),
> а лицом продукта становится **лента вакансий**.

## Идея

Сайт, где собраны ~**1000 Telegram-каналов с вакансиями**. По каждому каналу
**ежедневно подтягивается статистика** (подписчики, охваты, активность — уже умеем).
Из постов этих каналов **извлекаются вакансии** и попадают в **единую ленту**.

Три ключевых экрана:

1. **Лента вакансий** — общий поток вакансий из всех каналов. Фильтры
   (профессия/направление, формат — удалёнка/офис/гибрид, занятость, локация,
   зарплатная вилка, язык, источник-канал). Листается как лента; у каждой карточки
   видно **откуда она** (какой канал/сайт) и ссылка на оригинал.
2. **Каналы** — каталог отслеживаемых job-каналов с дневной статистикой
   (подписчики, рост, охват, ER, сколько вакансий дал канал). Отдельная страница
   на канал. Это текущий дашборд, переосмысленный под «качество источника».
3. **Сайты поиска работы** — отдельные страницы-агрегаторы по площадкам
   (hh, Хабр Карьера, Геекджоб, LinkedIn и т.п.): вакансии и метрики по каждому
   источнику; те же фильтры.

## Зачем (ценность)

- Вакансии в Telegram размазаны по сотням каналов — мы собираем их в один
   фильтруемый поток.
- Статистика каналов = **сигнал доверия**: видно живой канал или мёртвый, какой
   охват, как часто публикует работу.
- Единый формат карточки вакансии вместо разнородных постов.

## Модель данных (расширение текущей схемы)

Текущее (`schema.sql`): `channels`, `channel_snapshots`, `posts`, view `channel_latest`,
`watchlist`. Добавляем поверх:

- `channels.kind` — тип источника: `telegram_channel` | `job_site`.
- `channels.category` — направление (IT, дизайн, маркетинг, …) — уже есть колонка.
- **`jobs`** — извлечённые вакансии (одна строка = одна вакансия):
  - `id`, `source_channel_id` → `channels.id`, `source_post` (channel_id+message_id),
    `posted_at`, `title`, `company`, `description`,
    `profession` / `category`, `seniority`, `employment` (full/part/contract),
    `work_format` (remote/office/hybrid), `location`, `salary_min`, `salary_max`,
    `currency`, `tags[]`, `apply_url`, `raw_text`, `lang`, `created_at`.
  - Заполняется на этапе парсинга из `posts` (regex/эвристики → позже LLM-извлечение).
- View **`jobs_feed`** — вакансии + данные источника (название канала, фото, ER)
  для ленты одним запросом.

Парсер (`parser/parse_channels.py`): после сбора постов прогоняет их через
экстрактор вакансий и upsert-ит в `jobs`. Сайты работы — отдельные коннекторы
(их API/парсинг), пишут в те же `jobs` с `kind=job_site`.

## Прототип

`prototype.html` — статичный кликабельный прототип (mock-данные, без бэкенда):
**светлая тема, без эмодзи** (все иконки — SVG), солидный многораздельный сайт в
духе tg-stat. Верхняя навигация: Лента вакансий / Каталог каналов / Сайты работы /
Рейтинг. В ленте — фильтры (направление, формат, занятость, грейд, тип источника,
зарплата) и переключение «список / плитка»; в каталоге — сайдбар категорий и
сортируемая таблица. Цель — согласовать UX до доработки схемы и парсера.
